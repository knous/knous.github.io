前缀和+map

这个办法很有意思，先来看看 LeetCode 官方的解释：

它说的可能很复杂，我解释一下。

假设数组是：1，2，3

此时我们逻辑走起来，map<int,int> 里面要存放一个值，<0,1>，为什么要存这个，因为我们找的值是前缀和 - k

举个例子，这里我要找的是 3 ，那我的前缀和就是，0，1，3，6。

此时逻辑开始走，现在在前缀和里面开始走，0 - 3 = -3，那我就要在 map 里面找 -3 有没有，如果没有，说明这个位置肯定不会是符合 k 的区间，于是我在 map 里面加一个 <-3，1>。

继续往下走，当我拿到这个 3 的时候，我就在 map 里面找 3 - 3 = 0，这里就说明为什么了要有 <0，1>，因为只有这样你之后找的时候这里才不会出现明明等于 k 但是没算上。

此时计数就要 ++。

然后我们把 3 加到 map 里面 <3，1>。

继续往后走，6 - 3 = 3，此时再去 map 里面找 3 ，找到了就继续++。

如果上面逻辑听懂了，那之后的代码你也能看懂。

不过这里要注意的是，计数不是单纯的每次 ++，而是 + 你要找的值后面的那个计数

比方说这样，一开始的 map 是 <0，1>。

1 - 0 = 1，没找到，就是在 map 后面多加一个 <1,1>。

然后重点就来了，现在它到了 -1，-1+1 = 0，然后它去找，找到了这个 <0，1>，然后 count ++，但是下一个又是 0，0 - 0 = 0，它再去找 0 的时候，<0，1>，它再++，变成 2 ，因为当前缀和 等于 tar 的时候，说明上一个位置到这一个位置（1，-，0）是满足条件的，而这个位置本身也是满足条件的，因为有<0，2>说明当 tar 为 0 的时候有两个位置都满足条件，所以不能只加 1，而是要加 对于的位置数。
